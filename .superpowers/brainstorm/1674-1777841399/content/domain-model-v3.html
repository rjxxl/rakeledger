Note on staff_advance and fnb_cost: These are real cash leaving the drawer that isn't matched by an asset on the books — they're expenses. We model the "other side" as an implicit expense bucket (or call it operating_expense if we want it tracked). For v1 it's enough that the cashier records "${X} taken out for [reason]" so the drawer count doesn't appear short.
