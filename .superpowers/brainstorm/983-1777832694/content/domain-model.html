Section 2 — Domain Model

Boxes are entities (database tables). Arrows show how they relate.

Core ledger entities

Session 🌙

  • PK id
  • opened_at timestamp
  • closed_at timestamp?
  • opened_by → User
  • closed_by → User?
  • opening_cash decimal
  • closing_cash decimal?
  • notes text?

Transaction 💸

  • PK id
  • session_id → Session
  • type enum
  • created_at timestamp
  • created_by → User
  • player_id → Player?
  • staff_id → User?
  • table_id → Table?
  • amount decimal
  • method enum (cash/zelle/...)
  • note text?
  • reverses_id → Transaction?

LedgerEntry 📒

  • PK id
  • transaction_id → Transaction
  • account enum
  • delta decimal (+/-)
  • created_at timestamp
People

User 👤

  • PK id
  • name
  • email
  • role enum (owner/cashier/host/runner/dealer)
  • pin_hash string
  • active bool

Player 🃏

  • PK id
  • display_name
  • phone string?
  • notes text?
  • credit_limit decimal
  • created_at timestamp

Table 🪑

  • PK id
  • name (e.g. "Table 1")
  • game_type enum
  • stakes string
  • active bool
Specialized

Marker 🏦

  • PK id
  • player_id → Player
  • issued_in_session → Session
  • issued_tx → Transaction
  • amount decimal
  • repaid_amount decimal
  • status enum (open/repaid/written_off)

SessionAccountClose 🔒

  • PK id
  • session_id → Session
  • account enum
  • expected decimal (computed)
  • counted decimal (entered)
  • variance decimal
  • note text?

RakeDistribution 💰

  • PK id
  • session_id → Session
  • recipient_user_id → User
  • amount decimal
  • tx_id → Transaction

How it all fits together

  • A Session brackets the night. Everything else lives inside one.
  • A Transaction is what the user sees and records ("Player X buys in $500 via Zelle"). Has a type (buy_in, cash_out, rake, tournament_fee, tournament_payout, tip_payout, marker_issue, marker_repay, jackpot_payout, staff_advance, fnb_cost, drawer_count_adjust).
  • Each Transaction generates exactly two LedgerEntry rows that sum to zero (double-entry). account is one of: cash_drawer, zelle, venmo, cashapp, chip_float, marker_outstanding, rake_pool, tip_pool, tournament_pool.
  • Mistakes are corrected by adding a new Transaction with reverses_id set — never UPDATE or DELETE the original.
  • SessionAccountClose is the end-of-night reconciliation row per account: what the system says the balance should be vs. what was physically counted.
